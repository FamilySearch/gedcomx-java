  Gedcomx
    2.66.0  The last version before any generic naming changes were made
    2.67.0  Generic names are defined in the model and both names are available in the model and in the SDK
    2.71.0  Old naming support from the SDK has been removed.
                  The SDK (GedcomxApplicationState) always adds a header "X-FS-Feature-Tag" "generic.relationship.terms" to platform-API calls
                  FamilySearchFamilyTree.addChildAndParentsRelationshipNew() is the method name for now
            The model has both new and old deprecated names available.
                  ChangeObjectType  ChangeType  ChildAndParentsRelationship  RelationshipRole
            Utility classes have updates
                  FamilySearchPlatformModelVisitorBase visits both new and old deprecated fact field names automatically
                  FamilySearchPlatform - method name changes
                        findChildAndParentsRelationship() -> findChildAndParentsRelationshipNew() and findChildAndParentsRelationshipOld()
                        addParentChildRelationshipForEachChildAndParentsRelationship() ->
                                            addParentChildRelationshipForEachChildAndParentsRelationshipNew() and
                                            addParentChildRelationshipForEachChildAndParentsRelationshipOld()
                        fixLocalReferences() -> fixLocalReferencesNew() and fixLocalReferencesOld()
                            However fixLocalReferences() overrides a method in it's super class Gedcomx, so the "unnamed" method still exists and
                            just calls fixLocalReferencesNew() by default.  Users will need to search for this rather than rely on the compiler.
                  When using addParentChildRelationshipForEachChildAndParentsRelationshipNew local Relationship ids changed from "F"+id and "M"+id
                      to "P1"+id and "P2"+id.  These did not change for addParentChildRelationshipForEachChildAndParentsRelationshipOld()
    "remove-deprecated" branch (currently 2.72.0.SNAPSHOT) exists where deprecated model and utilities have been removed.  However the SDK still
            calls platform-API with header.  This can be used to find locations where the old deprecated model was used.
